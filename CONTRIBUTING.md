# Contributing to a language fork

This file is a placeholder. When you fork this template for a specific
language, replace it with instructions in that language so new
contributors know how to help.

Real translation teams typically cover:

- How to reserve a file (open an issue, claim a file in a spreadsheet,
  etc.) so two people do not translate the same catalog.
- How to set up the toolchain: installing `requirements-dev.txt`,
  installing [Poedit](https://poedit.net/) or another `.po` editor,
  running `make` to build the HTML locally.
- Translation conventions for the language: terminology glossary,
  formal vs. informal address, treatment of code identifiers, typographic
  rules (quotes, non-breaking spaces, etc.).
- Review expectations: how many reviewers, whether `fuzzy` entries need
  explicit sign-off, whether wrapping with `powrap` is mandatory.
- Where to discuss (Discord, mailing list, Discourse category, ...).

See the CONTRIBUTING files in the existing translations for examples:

- [python/python-docs-fr](https://github.com/python/python-docs-fr/blob/3.13/CONTRIBUTING.rst)
- [python/python-docs-tr](https://github.com/python/python-docs-tr/blob/3.12/README.md)
